📋 模块零：HC 业务需求审批

使用场景：业务主管需要招人，HR 需要审核需求合理性。告别口头沟通，实现需求线上化流转。

1 业务线主管（提报）

  1. 进入模块零的 [📤 我是业务：提报新 HC] 页面。
  2. 用大白话填入要解决的核心麻烦（Mission）、红线（Deal Breakers）等信息，点击提交。

2 HRBP（审批）

  1. 切换到 [✅ 我是 HR：审批 HC 需求] 页面。
  2. 查看业务线提交的工单，点击蓝色的"批准"或红色的"驳回"。批准后的 HC 会自动流入下一个模块。

🎯 模块一：JD 逆向与自动寻源引擎

痛点解决：拒绝"输入垃圾，输出垃圾"。一键生成顶级英文 JD 与极客级猎头搜索词。

1 承接业务需求

在页面最上方的下拉菜单中，选择刚才在"模块零"中已经批准的 HC。系统会瞬间将业务主管的大白话需求填充到下方表单中。

2 获取双重交付物

  • 高转化率 JD：突出 Alauda 战略高度的职位描述。
  • Google X-Ray 搜索代码 (Boolean Strings)：系统为你生成针对 LinkedIn, GitHub 的高级搜索代码，让你实现 10 倍搜索效率。

✉️ 模块二：自动化触达 (Cold Outreach)

痛点解决：抛弃废话模板，用高度个性化的营销级话术直击候选人痛点，大幅提升高端人才回复率。

1 输入候选人情报

系统自动读取模块一生成的职位信息，你只需补充候选人的名字和在领英上的核心亮点（如曾主导过什么项目）。

2 生成破冰话术

一键生成结构严谨的长篇邮件（Email版）和短小精悍的私信（InMail版）。

📄 模块三：猎头简历智能雷达 (Resume Matcher)

痛点解决：解决 HR 看不懂海外技术简历、容易被候选人过度包装忽悠的问题。

1 确定测量标尺

左侧确认本次招聘职位的核心 JD（系统会自动继承模块一的生成结果）。

2 批量上传简历

在右侧点击或拖入多份猎头/内推发来的简历文件，支持 PDF、DOCX（Word）、TXT 三种格式，可同时上传多份批量评估。

3 获取硬核体检报告

点击"硬核评估"，AI 会基于严苛的【算分卡法则】进行防漂移打分。它将输出：

  • 综合匹配度打分（0-100）
  • 核心亮点
  • 🚨 绝对红线预警
  • 🎯 破冰追问建议
💡 防漂移机制：AI 面试官在后台执行 Temperature=0.0 的绝对客观推理，将定性评估转为纯粹的"维度减法"，彻底消除大模型的"打分漂移（Evaluation Drift）"。

📝 模块四：结构化面试评估系统

痛点解决：统一全球各部门面试官的评分标准，消除面试随机性和个人偏见。

1 生成评估卡

承接前序生成的 JD，系统会自动拆解出"技术胜任力、售前交付力、文化契合度"三个核心维度。

2 分发给考官

系统会输出一份带 1-5 分严格定义标准的【行为锚定评分卡 (BARS)】以及对应的 STAR 深度追问题库。

📚 模块五与六：知识库自生长与 RAG 智库

痛点解决：打破出海政策"黑盒"，解决初创出海团队没有现成大部头知识库的冷启动问题。

🏗️ 模块六：录入与自动爬虫

HR 可通过选择内置的"普华永道/四大律所"合规指南链接，让 AI 自动抓取网页并提炼精华情报（自动清洗反爬虫限制）；或手工记录零散踩坑经验。点击"编译"，系统会自动排版生成精美的卡片式知识库。

  • 去重保护：重复爬取同一网页时系统自动识别并跳过，知识库不会因误操作膨胀。
  • 时效管理：每条情报默认有效期 90 天。临近到期的条目以琥珀色边框标注，已过期的以红色标注，提醒 HR 及时用最新数据更新。

📚 模块五：问答

在这里向 AI 提出出海相关的政策、薪酬问题。AI 会基于官方静态 PDF 与大家共建的动态 Playbook 进行检索回答，并提供精准的溯源。

💡 建议顺序：先用模块六构建知识库，再用模块五进行智能问答，效果更佳。

👥 模块七：候选人 Pipeline 看板

痛点解决：结束用表格或微信追踪候选人的混乱局面，将所有在途候选人纳入统一可视化漏斗管理。

1 添加候选人

点击"➕ 添加新候选人"，填写姓名、目标岗位、LinkedIn 主页链接，并可关联具体的 HC 需求 ID，候选人自动进入 Sourced（已发现）阶段。

2 看板追踪进展

所有在途候选人以 Kanban 卡片形式分列展示，7 个阶段各以独立颜色区分：

  • Sourced（已发现） → Contacted（已触达） → Phone Screen（电话初筛）
  • Interview（面试中） → Offer（发 Offer） → Hired（已入职）/ Rejected（淘汰）

每张卡片下方有阶段下拉菜单，选中目标阶段点击"移动"即可一键推进，操作历史自动记录时间戳。

3 管理候选人详情

在页面下方的"候选人详情"面板中可追加备注（每条自动打上日期）、查看完整阶段流转记录，以及查看关联的简历评分结果。

💡 工作流建议：从模块一生成 X-Ray 搜索词 → 在 LinkedIn 找到候选人 → 在模块七录入并标记 Sourced → 发送模块二的外联邮件后更新为 Contacted → 模块三评分后填入 Score → 进入面试阶段后继续推进，实现全链路闭环。

📊 招聘数据看板

全局数据可视化：漏斗转化率、渠道 ROI、Token 消耗追踪

1 KPI 总览

页面顶部横幅实时展示四大核心指标：开放 HC 数、在途候选人、已入职人数、平均简历评分。

2 可视化图表

  • 招聘漏斗转化：柱状图展示各阶段候选人数量及阶段间转化率。
  • 来源渠道分布：按来源渠道统计候选人数量及入职效率（ROI）。
  • 简历评分分布：四档分布统计（<60 淘汰 / 60-79 边缘 / 80-89 通过 / 90+ 优秀）。
  • HC 需求地区分布：按地区统计岗位需求量。

3 LLM Token 消耗追踪 V2.0 新增

页面底部新增"近期 LLM 调用记录"面板，实时展示每次 AI 调用的模型名称、Prompt Tokens、Completion Tokens 和总消耗，帮助管理员监控 API 用量。

⚙️ 系统功能速查

常见操作与配置说明

访问密码

管理员可在 Streamlit Secrets 中配置 APP_PASSWORD 环境变量启用密码保护。未配置时系统对所有访问者开放。同一浏览器会话内无需重复输入密码。

JD 跨会话恢复

在模块一生成 JD 后，即使刷新或重新打开页面，模块二、三、四都会自动恢复上次生成的 JD 内容，无需重新生成。

强制刷新缓存

若遇到界面卡顿或数据显示异常，可点击左侧导航栏底部的"🔄 强制清理云端缓存"按钮。

Docker 部署 V2.0 新增

系统提供 Dockerfile，可通过 docker build -t recruitment-copilot . && docker run -p 8501:8501 --env-file .env recruitment-copilot 一条命令完成容器化部署。

数据存储

V2.0 起，系统数据从 JSON 文件迁移至 SQLite 数据库（data/recruitment.db，WAL 模式）。旧版 JSON 数据在首次启动时自动迁移，无需手动操作。
