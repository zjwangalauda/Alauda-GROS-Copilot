Alauda Global Recruitment OS (GROS) Copilot - 用户操作手册

欢迎使用 Alauda (灵雀云) 全球招聘智能体平台 (GROS AI Copilot)。本系统是一套可复制的全球精英人才获取操作系统，旨在帮助非技术背景的 HR 和出海业务线主管，实现从“作坊式招聘”到“流水线式精准捕获”的全面升级。

系统底层战略 (The Strategy)

本平台的核心灵魂不仅是 AI，更是一套固化在代码中的顶级猎头方法论： * 深耕业务：我们不看“职责”，只看“Mission (入职第一年要打下的仗)”。 * 特种部队标准：针对全球扩张（新加坡、马来西亚、南非等）与对标 OpenShift 的硬核挑战，寻找兼具云原生深度与 CXO 沟通能力的顶尖售前/交付架构师。 * 过程卡控：从 需求对齐(Calibration) -> 多渠道寻源(Sourcing) -> 自动化触达(Outreach) -> 结构化面试(Vetting) -> 决策(Decision) -> 复盘(Retro) 形成完整闭环。

核心功能操作指南

🎯 模块一：JD 逆向与自动寻源引擎

痛点解决：拒绝“输入垃圾，输出垃圾”。用四维拷问替代传统职责罗列，一键生成 JD 与极客级搜索词。

  1. 填写“精准画像拷问表”：
    • 1️⃣ 核心使命 (The Mission)：这个人入职第一年必须完成的 3 个关键任务是什么？
    • 2️⃣ 必须技术栈 (The Tech Stack)：Kubernetes, Docker, CI/CD 等。
    • 3️⃣ 绝对红线 (The Deal Breakers)：不能接受的特质。
    • 4️⃣ 核心卖点 (Selling Point)：为什么顶级人才要离开舒适区来 Alauda？
  2. 获取双重交付物：
    • 高转化率 JD：突出 Alauda 战略高度的职位描述。
    • Google X-Ray 布尔搜索代码 (Boolean Strings)：系统为你生成针对 LinkedIn, GitHub 的高级搜索代码，让你实现 10 倍搜索效率。

✉️ 模块二：自动化触达 (Cold Outreach)

痛点解决：抛弃“我们在招人，你有兴趣吗”的废话模板，用 Alex Hormozi 的营销框架直击候选人痛点，大幅提升高端人才回复率。

  1. 输入候选人情报：系统会自动读取模块一的职位信息，你只需输入候选人的名字和背景亮点（例如：曾在 Red Hat 工作，懂 Go 语言）。
  2. 生成破冰话术：点击生成，系统会输出：
    • Email 版：结构清晰、极具煽动性的英文长信。
    • LinkedIn InMail 版：控制在 300 字以内的精简版本，适合手机端碎片化阅读。

📝 模块三：结构化面试评估系统

痛点解决：统一全球面试官标准，确保“识人”准确率，消除面试随机性。

  1. 输入 JD 文本：直接导入模块一生成的 JD。
  2. 生成行为锚定评分卡 (BARS Scorecard)： 系统会自动拆解出以下核心维度，并定义 1分（不合格）、3分（合格）、5分（卓越）的具体表现：
    • 技术胜任力 (Technical Competency)
    • 售前/交付能力 (Consulting/Delivery)
    • 文化契合度 (Culture Add)
  3. 获取 STAR 面试题库：每个维度都会附带 2-3 道深度追问的 STAR 行为面试题。

📚 模块四与五：知识库自生长与 RAG 智库

痛点解决：打破出海政策“黑盒”，解决“没有现成知识库”的冷启动问题。

  • 模块五（录入）：HR 可以在实战中，将零散的踩坑经验（如某国签证最低薪资、某地劳动法隐坑）录入系统。点击编译后，系统会动态生成属于 Alauda 自己的《全球招聘实战 Playbook》。
  • 模块四（问答）：基于静态 PDF 手册与刚才编译的动态 Playbook 进行智能检索与回答，并提供溯源能力，杜绝 AI 幻觉。

管理员配置说明 (IT/运维专用)

本系统核心由大语言模型与本地向量知识检索（FAISS）驱动。请修改项目根目录下的 .env 文件：

# 核心大脑（负责生成 JD、打分卡及聊天推理，已融入15年顶级猎头 Prompt）
OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
OPENAI_API_BASE=https://api.deepseek.com/v1

# 知识库引擎（负责将实战经验碎片向量化）
EMBEDDING_API_KEY=您的_兼容_OpenAI_格式的_Embedding_Key
EMBEDDING_API_BASE=对应的API地址
