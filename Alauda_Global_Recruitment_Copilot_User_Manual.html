Alauda Global Recruitment OS (GROS) Copilot - 用户操作手册

欢迎使用 Alauda (灵雀云) 全球招聘智能体平台 (GROS AI Copilot)。本系统是一套可复制的全球精英人才获取操作系统，旨在帮助非技术背景的 HR 和出海业务线主管，实现从“作坊式招聘”到“流水线式精准捕获”的全面升级。

🌐 系统访问

  • 云端免安装版地址: Alauda GROS Copilot (Streamlit Cloud) (建议将此链接加入浏览器收藏夹，支持电脑与手机端访问)

系统底层战略 (The Strategy)

本平台的核心灵魂不仅是 AI，更是一套固化在代码中的顶级猎头方法论： * 深耕业务：我们不看“职责”，只看“Mission (入职第一年要打下的仗)”。 * 特种部队标准：针对全球扩张（新加坡、马来西亚、南非等）与对标 OpenShift 的硬核挑战，寻找兼具云原生深度与 CXO 沟通能力的顶尖售前/交付架构师。 * 过程卡控：从 需求对齐(Calibration) -> 多渠道寻源(Sourcing) -> 自动化触达(Outreach) -> 初筛与面试(Vetting) -> 知识库沉淀(Playbook) 形成完整闭环。

核心功能操作指南

📋 模块零：HC 业务需求审批

使用场景：业务主管需要招人，HR 需要审核需求合理性。告别口头沟通，实现需求线上化流转。

  1. 业务线主管 (提报)：
  2. 进入模块零的 [📤 我是业务：提报新 HC] 页面。
  3. 用大白话填入要解决的核心麻烦（Mission）、红线（Deal Breakers）等信息，点击提交。
  4. HRBP (审批)：
  5. 切换到 [✅ 我是 HR：审批 HC 需求] 页面。
  6. 查看业务线提交的工单，点击蓝色的“批准”或红色的“驳回”。批准后的 HC 会自动流入下一个模块。

🎯 模块一：JD 逆向与自动寻源引擎

痛点解决：拒绝“输入垃圾，输出垃圾”。一键生成顶级英文 JD 与极客级猎头搜索词。

  1. 承接业务需求：在页面最上方的下拉菜单中，选择刚才在“模块零”中已经批准的 HC。系统会瞬间将业务主管的大白话需求填充到下方表单中。
  2. 获取双重交付物：
    • 高转化率 JD：突出 Alauda 战略高度的职位描述。
    • Google X-Ray 布尔搜索代码 (Boolean Strings)：系统为你生成针对 LinkedIn, GitHub 的高级搜索代码，让你实现 10 倍搜索效率。

✉️ 模块二：自动化触达 (Cold Outreach)

痛点解决：抛弃废话模板，用高度个性化的营销级话术直击候选人痛点，大幅提升高端人才回复率。

  1. 输入候选人情报：系统自动读取模块一生成的职位信息，你只需补充候选人的名字和在领英上的核心亮点。
  2. 生成破冰话术：一键生成结构严谨的长篇邮件（Email版）和短小精悍的私信（InMail版），均采用地道高级商务英语。

📄 模块三：猎头简历智能雷达 (Resume Matcher)

痛点解决：解决 HR 看不懂海外技术简历、容易被候选人过度包装忽悠的问题。

  1. 确定测量标尺：左侧确认本次招聘职位的核心 JD（系统会自动继承模块一的生成结果）。
  2. 批量上传简历：在右侧点击或拖入多份猎头/内推发来的 PDF 或 TXT 简历文件。
  3. 获取硬核体检报告：点击“硬核评估”，AI 会基于严苛的【算分卡法则】进行数学推导打分。它将输出：
  4. 综合匹配度总分及三维拆解 (使命契合度、技术栈硬实力、红线规避)
  5. 核心亮点与优势
  6. 🚨 绝对红线与造假预警
  7. 🎯 破冰追问建议 (供 HR 电话沟通时使用)

📝 模块四：结构化面试评估系统

痛点解决：统一全球各部门面试官的评分标准，消除面试随机性和个人偏见。

  1. 生成评估卡：承接前序生成的 JD，系统会自动拆解出“技术胜任力、售前交付力、文化契合度”三个核心维度。
  2. 分发给考官：系统会输出一份带 1-5 分严格定义标准的【行为锚定评分卡 (BARS)】以及对应的 STAR 深度追问题库。建议在面试前下载 Markdown 发送给所有考官。

📚 模块五与六：知识库自生长与 RAG 智库

痛点解决：打破出海政策“黑盒”，解决初创出海团队没有现成大部头知识库的冷启动问题。

  • 模块六（录入）：HR 可随时将零散的踩坑经验（如某国工作签的潜规则）用大白话录入系统。点击“编译”，系统会自动排版生成企业专属的动态 Playbook。
  • 模块五（问答）：在这里向 AI 提出出海相关的政策、薪酬问题。AI 会基于官方静态 PDF 与大家共建的动态 Playbook 进行检索回答，并提供精准的溯源。

OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
OPENAI_API_BASE=https://api.deepseek.com/v1

EMBEDDING_API_KEY=您的_兼容_OpenAI_格式的_Embedding_Key
EMBEDDING_API_BASE=对应的API地址
